LESSON 01 · BEATS

A pattern is a string of time.

Type a rhythm between quotes, route it into a drum with ->, and it plays. A 1 is a hit, a ~ is a rest. The line reads left to right, exactly like the music: source → instrument → speakers.

Toggle the steps below. The fugue code rewrites itself to match what you hear.

LESSON 02 · SOUNDS

        
Build an instrument with one pipe.

        
Drums are processes someone already wrote. Here's how they're made. A process is a signal flowing through functions — the |> pipe sends a raw saw wave through a filter and an envelope until it's an instrument.

        
The knobs are the parameters in the code. Turn them — hear a buzzy saw become a soft pluck or a screaming lead. This is the part Tidal doesn't have: the sound is yours to shape.

Low cutoff + high resonance = the classic acid squelch. Short decay = a pluck.

      


      
      

        
LESSON 03 · BASSLINES

        
Numbers become notes.

        
Put note names in the pattern instead of 1s and route them to a voice — now the same instrument plays pitches. Click a step to cycle it through an A-minor pentatonic scale, or back to a rest.

        
It's the same -> as the drums. A pattern aimed at a voice's input becomes notes; the voice's job is just to make each one sound.

LESSON 04 · CHORDS

        
One chord, many voices.

        
Write a chord by name and Fugue knows its notes. The |> pipe sends it through voicing to spread it nicely, and routing it to a process plays that process once per note at the same time — that's polyphony, for free.

        
Pick a chord for each half of the bar. The progression loops; the same keys instrument voices all of them.

LESSON 05 · MELODIES

        
Bend a line with combinators.

        
A melody is just a higher pattern — but outside the quotes you can transform it with |>, the same pipe you used on chords. rev plays it backwards; jux(rev) sends the original to the left ear and a reversed copy to the right; every(4, rev) flips it only on the fourth bar, so it never quite repeats.

        
Write the notes once. Let the combinator keep them alive.

Try jux(rev) and listen wide — the same notes, mirrored across the stereo field.

      


      
      

        
LESSON 06 · MOVEMENT

        
Wire one thing into another.

        
This is the part no pattern language has. The score is a patchbay: any signal can drive any parameter. The kick routed into pad.duck ducks the pad on every hit — sidechain, as one line of wiring. Inside the pad, an lfo breathes the filter cut so it never sits still.

        
Turn the LFO up. Toggle the sidechain. The pad never stops moving.

Slow LFO + deep sidechain = the whole "pumping, evolving" feel of dub techno, from two wires.

      


      
      

        
LESSON 07 · ARRANGE

        
A song is a grid of on and off.

        
Fugue has no "section" keyword — and doesn't need one. Multiply a voice by a slow pattern of 1s and 0s and it plays only in those bars. The grid below is the arrangement: each row a voice, each column a bar. Toggle cells to bring parts in and out across eight bars.
