Shape a patch = processes + a score

tempo = 114bpm

process bass(note, trig) {                 // a signal graph
    cut = param.log(60Hz, 700Hz) 180Hz |> smooth(10ms)   // a knob, de-zippered
    out = (saw_naive(note) |> ladder(cut, 0.4)) * ar(6ms, 240ms, trig)
}

"<Bbm9 Fm9>" |> n("0 ~ 4 ~") |> octave(-1) -> bass -> out
  • out: out = … is the audible output — no out ⇒ silent; out = (l, r) ⇒ stereo.
  • ports: pitch & gate are separate — process v(note, trig); the trigger port must be named trig; audio in needs in: audio.
  • note: in the body, note is a frequency in Hz — note*2 = octave up.
  • params: cut = param.log(lo, hi) default (also .lin .exp .step); drive with val -> v.cut.
  • poly: explicit — process pad(note) voices 6 { }.
  • feedback: feedback loop = (in + loop*fb) |> delay(375ms).

Operators loosest → tightest

->connect source → target (the score)
|>compose / transform, left → right (DSP body + pattern transforms)
+ -add, subtract
* /multiply, divide
  • |> threads into the first arg — x |> lpf(fc, q) ⇒ lpf(x, fc, q).
  • |> binds loosest — wrap a pipe before */+: (x |> lpf(fc, q)) * env.

Routing the score — source -> target lines

"pat" -> voicefeed a pattern into a voice (note / trigger)
"pat" -> voice.cutfeed a param (dotted = the param's own name)
lfo(0.2Hz) -> v.cuta signal drives a param
kick -> bass.duckvoice out → another's audio port (sidechain)
bass -> reverb -> outroute through an effect, then to the master
v |> pan(0.3) -> outpanned send (p ∈ −1…1)
v * "<1 0>" -> outgated send = arrangement / mute
  • Feed and output are separate lines. Feed once ("pat" -> bass), then reference by name for output (bass -> out, bass |> pan(x) -> out). A voice may fan out to many consumers.
  • One send form at a time — plain v -> out, panned v |> pan(p) -> out, or gated v * "<1 0>" -> out. Gating multiplies the audio output, not the note pattern.
  • lfo(hz) is a bipolar sine ±1 — drive a param as 800Hz + lfo(0.2Hz)*400Hz.

Stdlib stages |> = piped signal is the first arg

  • Args are dimensioned — arg:Hz a frequency (8kHz), arg:s a time (220ms), arg:dB a level (-12dB).
  • A bare number is a dimensionless Real and won't coerce — write plate(550ms, 0.3), not plate(0.55, …).
  • Unmarked args (Q, mix, depth, drive, ratio, sustain, fb) are bare Reals. Inside a quote it's the reverse — numbers are bare.

Patterns mini-notation inside quotes · one quote = one cycle = one bar

1 2 3sequence — steps split the cycle evenly
~rest (a silent step)
a _tie — hold the previous step across this one
[a b]subdivide one step into a sub-sequence
<a b>alternate — one per cycle
a*Nreplicate the step N× inside its slot
a?drop the step ~50% of cycles (a?p for prob p)
  • Value atoms are bare numbers — no unit suffix in a quote: "<200 600>" -> v.cut, never "<200Hz 600Hz>" (the target supplies the unit).
  • At a trigger port a token fires on presence — 1 = hit, ~ = rest. Adjacent onsets re-strike; _ ties them legato.
